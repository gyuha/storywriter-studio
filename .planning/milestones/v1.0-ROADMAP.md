# Milestone v1.0: StoryWriter Studio MVP

**Status:** ✅ SHIPPED 2026-05-26
**Phases:** 1-4
**Total Plans:** 13

## Overview

기존 FastAPI DDD + React 19 + LangChain/LiteLLM 스택 위에 소설 집필 특화 기능을 4개 Phase에 걸쳐 구축했다. 인증 연동으로 기반을 확립하고, 소설/챕터 관리로 콘텐츠 구조를 만들고, 세계관 데이터베이스로 AI 컨텍스트의 원천을 구축한 뒤, 사이드패널과 AI 초안 생성으로 핵심 가치를 실현했다.

## Phases

### Phase 1: 인증 연동

**Goal:** 사용자가 실제 백엔드 JWT로 인증하여 소설 도메인에 안전하게 접근할 수 있다
**Depends on:** Nothing (first phase)
**Plans:** 4 plans

Plans:
- [x] 01-01-PLAN.md — 백엔드 관리자 API (admin_router.py, admin_schemas.py, DB seed migration)
- [x] 01-02-PLAN.md — 프론트엔드 auth 타입/API 레이어 (types/auth.ts 확장, auth-api.ts 생성, mock 삭제)
- [x] 01-03-PLAN.md — 프론트엔드 인증 연결 (use-init-auth, use-auth-mutation 교체, _authenticated.tsx, __root.tsx)
- [x] 01-04-PLAN.md — 관리자 사용자 관리 UI (AdminUsersPage, admin 훅, 라우트)

**Success Criteria:**
1. 사용자가 이메일/비밀번호로 로그인하면 실제 FastAPI 백엔드 JWT를 발급받고 세션이 유지된다
2. 사용자가 회원가입하면 즉시 계정이 생성되고 로그인 상태로 전환된다
3. 인증되지 않은 사용자가 소설 프로젝트 URL에 접근하면 로그인 페이지로 리다이렉트된다
4. 관리자가 관리자 페이지에서 사용자 목록 조회 및 계정 활성화/비활성화가 가능하다

---

### Phase 2: 소설/챕터 관리

**Goal:** 사용자가 소설 프로젝트를 생성하고, 챕터를 추가하여 TipTap 에디터로 본문을 작성할 수 있다
**Depends on:** Phase 1
**Plans:** 4 plans

Plans:
- [x] 02-01-PLAN.md — 백엔드 novel 도메인 (DB 마이그레이션, models/schemas/repository/service/router, 단위 테스트)
- [x] 02-02-PLAN.md — 프론트엔드 소설 CRUD (features/novel/ 타입·API·훅·컴포넌트, /novels 및 /novels/:id 라우트)
- [x] 02-03-PLAN.md — 챕터 에디터 (TipTap 설치, ChapterEditor, 자동저장 훅, 글자수·저장 상태 표시)
- [x] 02-04-PLAN.md — 챕터 목록 사이드바 (dnd-kit SortableList, 순서 변경, 상태 변경, ChapterSidebar 연결)

**Success Criteria:**
1. 사용자가 소설 프로젝트를 생성·수정·삭제하고 목록에서 전체를 확인할 수 있다
2. 챕터를 드래그앤드롭으로 순서 변경, 상태(초안/검토 중/완성) 변경 가능
3. TipTap 에디터에서 굵게/이탤릭/단락 스타일 작성, debounce 자동저장
4. 에디터 하단에 현재 글자 수 실시간 표시, 5,000자 목표 달성 시 시각적 피드백

### Phase 3: 세계관 데이터베이스

**Goal:** 사용자가 소설의 캐릭터·장소·세계관 설정을 구조화된 데이터베이스로 관리할 수 있다
**Depends on:** Phase 2
**Plans:** 3 plans

Plans:
- [x] 03-01-PLAN.md — DB 스키마 + 백엔드 도메인 (5개 테이블 마이그레이션, domains/world/ 전체, HeyAPI 재생성)
- [x] 03-02-PLAN.md — 세계관 관리 프론트엔드 (features/world/ 타입·API·훅·컴포넌트, 캐릭터·장소·세계관설정 탭)
- [x] 03-03-PLAN.md — 타임라인 + 캐릭터 관계 (백엔드 timeline/relationship router, 시간표 탭, 캐릭터 관계 서브섹션)

**Success Criteria:**
1. 캐릭터 생성·수정·삭제, 다른 캐릭터와의 관계 정의 가능
2. 장소와 세계관 설정 항목 생성·수정·삭제 가능
3. 시간표/연표 항목 생성 및 챕터 연결 가능
4. 세계관 항목 이름/유형으로 검색·필터링 가능

### Phase 4: 에디터 사이드패널 + AI 초안 생성

**Goal:** 작가가 챕터를 편집할 때 캐릭터·장소·설정이 자동으로 AI 컨텍스트에 포함되어 세계관과 일관된 챕터 초안을 AI가 생성한다
**Depends on:** Phase 3
**Plans:** 2 plans

Plans:
- [x] 04-01-PLAN.md — 사이드패널 레이아웃 + AI 초안 생성 (2-패널 레이아웃, 컨텍스트 토글, SSE 스트리밍, draft_router)

**Success Criteria:**
1. 챕터 에디터가 좌측 편집 영역 + 우측 사이드패널 2-패널 레이아웃으로 표시
2. 사이드패널에서 개별 항목을 "AI 컨텍스트에 포함" 토글로 선택/제외 가능
3. "AI 초안 생성" 버튼 클릭 시 SSE로 에디터에 실시간 스트리밍, 생성 중 read-only + 취소 버튼 표시

## Milestone Summary

**Key Decisions:**

- HeyAPI 기반 OpenAPI SDK 자동 생성 채택 — mock-auth-api.ts 완전 제거, 실제 백엔드 연결
- TipTap 3.x 에디터 채택 — React 19 호환, 즉각적 하이드레이션 비활성화 필요 (`immediatelyRender: false`)
- 챕터 순서 관리 — fractional indexing (float order_key) + dnd-kit SortableList
- 세계관 스키마 — 정규화 컬럼 + JSONB 하이브리드 (세계관 설정 자유 형식 지원)
- AI 컨텍스트 전략 — MVP는 Structured Prompt Injection, 항목 200개 이상 시 pgvector RAG 전환 예정
- draft_router — novel 도메인에서 langchain_core.messages 직접 사용 (chat 도메인 의존성 없이 구현)

**Issues Resolved:**
- auth mock 제거 후 HeyAPI 생성 SDK로 완전 교체 완료
- TipTap React 19 hydration 오류 → `immediatelyRender: false` 적용으로 해결
- 세계관 관계 그래프 — GraphEdge.type과 RelationshipType 타입 충돌 해결

**Issues Deferred:**
- AI-04 (AI 모델 선택 UI): LiteLLM 기반 다중 모델 선택 — Phase 4 MVP에서 제외, v2로 이동
- AI-05 (이전 챕터 요약 자동 포함): 컨텍스트 연속성 기능 — v2로 이동
- pgvector 기반 의미 검색 (ADV-03): 항목 200개 초과 시 활성화 예정

**Technical Debt Incurred:**
- draft_router.py — langchain_core를 novel 도메인에서 직접 임포트 (LLM 격리 원칙 일부 완화)
- 요구사항 체크리스트 — REQUIREMENTS.md 체크박스 업데이트 미완료 (문서 갈등)
- Phase 4 SUMMARY.md 미생성 — gsd-sdk SUMMARY 형식 문서화 누락
