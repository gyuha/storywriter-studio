## Objective

Plan 1에서 생성된 백엔드 API와 HeyAPI 재생성 결과를 바탕으로 세계관 관리 프론트엔드를 구현한다.
캐릭터·장소·세계관 설정 3개 엔티티의 CRUD UI를 `/novels/:novelId/world` 페이지에 탭 형태로 제공한다.

Purpose: 사용자가 세계관 데이터를 브라우저에서 직접 생성·수정·삭제할 수 있어야 한다.
Output: `features/world/` 피처 디렉토리 전체, 라우트 파일 1개, 소설 상세 페이지 링크

## Tasks

### Task 1: 타입·스키마·API 레이어 + TanStack Query 훅

**Action:**

아날로그: `features/novel/types/novel.ts`, `lib/novel-api.ts`, `hooks/use-novel-queries.ts`, `hooks/use-novel-mutations.ts` 직접 참조.

1. `apps/web/src/features/world/` 디렉토리 구조를 생성한다:
   - `types/`, `schema/`, `lib/`, `hooks/`, `components/`

2. `features/world/types/world.ts` 작성 (아날로그: `features/novel/types/novel.ts`):
   - `id`, `novel_id`: `string` (UUID를 string으로)
   - datetime 필드: `string` 타입
   - optional 필드: `field?: string` 또는 `field: string | null`

   내보낼 인터페이스:
   ```
   Character { id, novel_id, name, appearance, personality, background, role, summary, created_at, updated_at }
   CharacterCreateInput { name: string; appearance?: string; personality?: string; background?: string; role?: string; summary?: string }
   CharacterUpdateInput { name?: string; ... (모든 필드 optional) }

   Location { id, novel_id, name, description, location_relation, summary, created_at, updated_at }
   LocationCreateInput { name: string; description?: string; location_relation?: string; summary?: string }
   LocationUpdateInput { 모든 필드 optional }

   WorldSettingType = 'magic_system' | 'nation_faction' | 'history' | 'rule'
   WorldSetting { id, novel_id, name, type: WorldSettingType, content: Record<string, unknown>, summary, created_at, updated_at }
   WorldSettingCreateInput { name: string; type: WorldSettingType; content?: Record<string, unknown>; summary?: string }
   WorldSettingUpdateInput { 모든 필드 optional }
   ```

3. `features/world/lib/world-api.ts` 작성 (아날로그: `features/novel/lib/novel-api.ts`):
   - `throwOnError` 헬퍼: `novel-api.ts` lines 10-18 그대로 복사
   - `apiGetCharacters(novelId, name?)`, `apiGetCharacter(novelId, characterId)`, `apiCreateCharacter(novelId, data)`, `apiUpdateCharacter(novelId, characterId, data)`, `apiDeleteCharacter(novelId, characterId)`
   - Location, WorldSetting 동일 패턴 함수 작성
   - HeyAPI 함수명은 `sdk.gen.ts`에서 실제 생성된 함수명을 확인하여 사용. 네이밍 패턴: `createCharacterApiV1NovelsNovelIdCharactersPost`, `listCharactersApiV1NovelsNovelIdCharactersGet` 등
   - `WorldSetting`의 경우 `apiGetWorldSettings(novelId, name?, type?)` — type 파라미터 추가

4. `features/world/hooks/use-world-queries.ts` 작성 (아날로그: `features/novel/hooks/use-novel-queries.ts`):
   queryKey 구조: `['worlds', novelId, 'characters']`, `['worlds', novelId, 'locations']`, `['worlds', novelId, 'world-settings']`

   ```typescript
   export function useCharacters(novelId: string, name?: string) {
     return useQuery({
       queryKey: ['worlds', novelId, 'characters', { name }],
       queryFn: () => apiGetCharacters(novelId, name),
       enabled: !!novelId,
     });
   }
   // useLocations, useWorldSettings 동일 패턴
   // useWorldSettings에 type 파라미터 추가
   ```

5. `features/world/hooks/use-world-mutations.ts` 작성 (아날로그: `features/novel/hooks/use-novel-mutations.ts`):
   `queryClient.invalidateQueries` + `toast.success` + `onError: toast.error` 패턴.

   각 엔티티별: `useCreate{Entity}Mutation`, `useUpdate{Entity}Mutation`, `useDelete{Entity}Mutation`
   - 성공 메시지: `'캐릭터가 추가되었습니다'`, `'캐릭터가 수정되었습니다'`, `'캐릭터가 삭제되었습니다'`
   - Location: `'장소가 ...'`, WorldSetting: `'세계관 설정이 ...'`

### Task 2: 캐릭터·장소·세계관 설정 컴포넌트 + WorldPage + 라우트

**Action:**

아날로그: `features/novel/components/novel-card.tsx`, `novel-create-modal.tsx` 패턴 직접 참조.

**모달 방식: 로컬 state** (RESEARCH.md §5.4 — novel 피처 패턴과 동일, D-19):
부모 컴포넌트가 `useState<boolean>(false)`로 모달 open 상태를 관리. `modal-store` 전역 방식 사용 안 함.

**삭제 confirm (D-21):** Radix Dialog 또는 `window.confirm`을 사용. 기존 컴포넌트에서 사용하는 방식과 일치시킨다.

**목록 UI: 테이블/리스트 (D-20):** 카드 그리드 아님.

**1. `character-form-modal.tsx` 작성:**
- Props: `{ open: boolean; onClose: () => void; novelId: string; character?: Character }`
- `character`가 있으면 수정 모드, 없으면 생성 모드
- `useForm` + `zodResolver` + zod 스키마 (컴포넌트 파일 상단에 인라인 정의):
  ```
  name: z.string().min(1, '이름을 입력해주세요').max(255)
  appearance, personality, background, summary: z.string().optional()
  role: z.string().max(100).optional()
  ```
- `useCreateCharacterMutation(novelId)` / `useUpdateCharacterMutation(novelId)` 사용
- `mutation.isPending`으로 제출 버튼 비활성화
- 성공 시 `onClose()` 호출
- UI: Dialog (shadcn/radix), Input, Textarea, Button

**2. `character-list.tsx` 작성:**
- Props: `{ novelId: string }`
- 로컬 state: `searchName: string` (입력값), `debouncedName: string` (쿼리 전달용)
  — debounce는 `setTimeout` 300ms 또는 별도 훅 없이 `useState`만 사용 (MVP)
- `useCharacters(novelId, debouncedName)` 호출
- 목록 UI: table (이름, 역할, 요약 컬럼) + 수정/삭제 버튼
- 수정 버튼: `setEditTarget(character)` + `setModalOpen(true)`
- 삭제 버튼: `if (window.confirm('삭제하시겠습니까?')) useDeleteCharacterMutation.mutate(characterId)`
- `isLoading` → 스피너, `isError` → 에러 메시지 (3-상태 패턴, PATTERNS.md)
- 우상단 '캐릭터 추가' 버튼으로 `CharacterFormModal` 열기

**3. `location-form-modal.tsx`, `location-list.tsx` 작성:**
- `character-form-modal.tsx` / `character-list.tsx`와 동일 패턴
- 필드: name(필수), description, location_relation, summary

**4. `world-setting-form-modal.tsx` 작성:**
- 추가 필드: `type` (select: magic_system/nation_faction/history/rule), `content` (textarea, JSON 텍스트)
- zod content 필드:
  ```typescript
  content: z.string()
    .default('{}')
    .refine((val) => { try { JSON.parse(val); return true; } catch { return false; } }, { message: '유효한 JSON 형식이 아닙니다' })
  ```
- 제출 시 `JSON.parse(formValues.content)`로 변환 후 API 전달
- 수정 모드 초기값: `JSON.stringify(character.content, null, 2)`

**5. `world-setting-list.tsx` 작성:**
- 로컬 state: `searchName`, `searchType: WorldSettingType | ''`
- `useWorldSettings(novelId, debouncedName, searchType || undefined)`
- 필터 UI: name 검색 input + type select (전체/magic_system/nation_faction/history/rule)
- 테이블 컬럼: 이름, 유형(한국어 표시), 요약, 수정/삭제

**6. `world-page.tsx` 작성 (아날로그: `$novelId/index.tsx` 구조):**
- Props: `{ novelId: string }`
- 탭 구현: shadcn Tabs 또는 로컬 state `activeTab: 'characters' | 'locations' | 'world-settings'`로 조건부 렌더링
- 탭 3개: '캐릭터' | '장소' | '세계관 설정'
- 각 탭 패널에 해당 list 컴포넌트 렌더링

**7. 라우트 파일 작성:**
- `apps/web/src/routes/_authenticated/novels/$novelId/world/index.tsx`:

  ```typescript
  import { createFileRoute } from '@tanstack/react-router';
  import { WorldPage } from '@/features/world/components/world-page';

  export const Route = createFileRoute('/_authenticated/novels/$novelId/world/')({
    component: WorldIndexPage,
  });

  function WorldIndexPage() {
    const { novelId } = Route.useParams();
    return <WorldPage novelId={novelId} />;

  ```

**8. 소설 상세 페이지에 세계관 링크 추가:**

- 기존 코드를 최소한으로 수정. 링크 위치는 챕터 목록 상단이나 탭 형태가 있다면 그 근처에 배치.
