## Objective

`domains/world/` 신규 도메인을 생성하고 캐릭터·장소·세계관 설정 3개 엔티티의 백엔드를 완성한다.
DB 마이그레이션(5개 테이블 전부), ORM 모델, Pydantic 스키마, repository·service·router 레이어까지 구현하고 `main.py`에 등록한다.
타임라인과 캐릭터 관계는 Plan 3에서 구현하지만, **ORM 모델과 마이그레이션은 이 Plan에서 전부 작성**한다(FK 의존성 때문).

Purpose: Plan 2(프론트엔드)가 HeyAPI로 자동 생성된 타입을 사용할 수 있도록 백엔드 API가 먼저 완성되어야 한다.
Output: 15개 CRUD 엔드포인트, Alembic 마이그레이션 1개, `domains/world/` 전체 구조

2. `apps/api/src/domains/world/models/world_models.py`를 작성한다.
   아날로그: `domains/novel/models/novel_models.py` — `novel_id FK + ondelete="CASCADE"`, `updated_at onupdate=lambda` 패턴 그대로 복사.

   **StrEnum 선언 (D-04):**
   - `WorldSettingType(enum.StrEnum)`: `MAGIC_SYSTEM="magic_system"`, `NATION_FACTION="nation_faction"`, `HISTORY="history"`, `RULE="rule"`
   - `RelationshipType(enum.StrEnum)`: `LOVER="lover"`, `ENEMY="enemy"`, `ALLY="ally"`, `FAMILY="family"`

   **ORM 모델 5개 (D-04):**

   `Character(__tablename__="characters")`:
   - `id`: UUID PK, `server_default=func.uuid_generate_v4()`
   - `novel_id`: UUID FK→novels.id, `ondelete="CASCADE"`
   - `name`: `String(255)`, not null
   - `appearance`, `personality`, `background`, `summary`: `Text`, nullable
   - `role`: `String(100)`, nullable
   - `created_at`, `updated_at`: `DateTime(timezone=True)`

   `Location(__tablename__="locations")`:
   - `id`, `novel_id` (동일 패턴)
   - `name`: `String(255)`, not null
   - `description`, `location_relation`, `summary`: `Text`, nullable
   - `created_at`, `updated_at`

   `WorldSetting(__tablename__="world_settings")`:
   - `id`, `novel_id` (동일 패턴)
   - `name`: `String(255)`, not null
   - `type`: `SAEnum(WorldSettingType, name="world_setting_type_enum", create_type=False)`, not null
   - `content`: `JSONB`, nullable=False (빈 경우 `{}`)
   - `summary`: `Text`, nullable
   - `created_at`, `updated_at`

   `Timeline(__tablename__="timelines")`:
   - `id`, `novel_id` (동일 패턴)
   - `event_name`: `String(255)`, not null
   - `event_date`: `String(100)`, nullable (자유 형식 VARCHAR, D-04)
   - `description`, `summary`: `Text`, nullable
   - `chapter_id`: UUID FK→chapters.id, `ondelete="SET NULL"`, nullable
   - `created_at`, `updated_at`

   `CharacterRelationship(__tablename__="character_relationships")`:
   - `id`: UUID PK
   - `novel_id`: UUID FK→novels.id, `ondelete="CASCADE"`
   - `character_id_a`, `character_id_b`: UUID FK→characters.id, `ondelete="CASCADE"`
   - `type`: `SAEnum(RelationshipType, name="relationship_type_enum", create_type=False)`, not null
   - `description`: `Text`, nullable
   - `created_at`, `updated_at`
   - UniqueConstraint 없음 (A→B와 B→A는 별개 레코드 허용, D-04)

   SAEnum `create_type=False` 이유: ENUM 타입은 마이그레이션에서 수동 생성한다(Pitfall 1 회피).

3. `apps/api/alembic/versions/0005_world_domain.py`를 작성한다.
   아날로그: `alembic/versions/0003_novel_domain.py` 전체 구조 복사.

   ```
   revision = "0005_world_domain"
   ```

4. `apps/api/src/domains/world/schemas/world_schemas.py`를 작성한다.
   아날로그: `domains/novel/schemas/novel_schemas.py` — `ConfigDict(from_attributes=True)`는 Response 클래스에만.

   각 엔티티별 Create / Update / Response 세트:

   `CharacterCreate`: `name: str`, `appearance/personality/background/role/summary: str | None = None`
   `CharacterUpdate`: 모든 필드 `str | None = None` (모두 optional)
   `CharacterResponse(ConfigDict from_attributes=True)`: `id: uuid.UUID`, `novel_id: uuid.UUID`, 나머지 필드, `created_at/updated_at: datetime`

   `WorldSettingUpdate`: `name/summary: str | None = None`, `type: WorldSettingType | None = None`, `content: dict | None = None`

### Task 2: Character + Location + WorldSetting — Repository, Service, Router

**Action:**

아날로그: `domains/novel/repository/chapter_repository.py`, `service/novel_service.py`, `router/novel_router.py` 패턴 직접 복사 + 수정.

**Repository 3개 작성:**

`character_repository.py` (아날로그: `chapter_repository.py`):
- `get_by_id(novel_id, character_id) -> Character | None`: `select(Character).where(Character.id == character_id, Character.novel_id == novel_id)`
- `list_by_novel(novel_id, name: str | None = None) -> list[Character]`: `name`이 있으면 `.where(Character.name.ilike(f"%{name}%"))` 추가 (D-13, WORLD-06)
- `create(novel_id, **fields) -> Character`: `flush()` + `refresh()` 쌍
- `update(character, **fields) -> Character`: `flush()` + `refresh()`
- `delete(character) -> None`: `session.delete()` + `flush()`

`location_repository.py`: 동일 패턴. `list_by_novel`에 `name` ILIKE 필터.

`world_setting_repository.py`: `list_by_novel`에 `name` ILIKE + `type: WorldSettingType | None = None` 정확 일치 필터 추가:
```python
if type is not None:
    stmt = stmt.where(WorldSetting.type == type)
```

**Service 3개 작성:**

각 서비스는 `novel_repo: NovelRepository`를 주입받아 소유권 검증 수행. 아날로그: `chapter_service.py`의 `_verify_novel_ownership`.

`character_service.py`:
- `__init__(self, novel_repo: NovelRepository, character_repo: CharacterRepository)`
- `_verify_novel_ownership(novel_id, user_id)`: novel 조회 → None이면 `NotFoundError("Novel")` → user_id 불일치면 `ForbiddenError()`
- `list_characters(novel_id, user_id, name?) -> list[Character]`: 소유권 검증 후 repo 위임
- `get_character(novel_id, user_id, character_id) -> Character`: 소유권 검증 → `NotFoundError("Character")`
- `create_character(novel_id, user_id, data: CharacterCreate) -> Character`: 소유권 검증 → `repo.create(novel_id, **data.model_dump(exclude_none=True))`
- `update_character(novel_id, user_id, character_id, data: CharacterUpdate) -> Character`: 소유권 검증 → get → `repo.update(..., **data.model_dump(exclude_unset=True))`
- `delete_character(novel_id, user_id, character_id) -> None`: 소유권 검증 → get → `repo.delete(...)`

`location_service.py`, `world_setting_service.py`: 동일 구조. `world_setting_service.py`의 `list_world_settings`는 `type` 파라미터 추가.

**Router 3개 + `world_router.py` 작성:**

엔티티별 라우터 파일 분리 (600줄 예상 → 파일 3개로 분리, RESEARCH.md §1.1 근거).

`character_router.py`:
```python
router = APIRouter(tags=["characters"])

async def _get_character_service(session: AsyncSession = Depends(get_async_session)) -> CharacterService:
    return CharacterService(NovelRepository(session), CharacterRepository(session))

def _app_error_to_http(error: AppError) -> HTTPException:
    return HTTPException(status_code=error.status_code, detail=error.message)
```
엔드포인트 5개 (D-08):
- `POST /novels/{novel_id}/characters` → 201, `response_model=CharacterResponse`

- `PUT /novels/{novel_id}/characters/{character_id}` → `response_model=CharacterResponse`
- `DELETE /novels/{novel_id}/characters/{character_id}` → 204, `response_class=Response`

모든 엔드포인트: `current_user: User = Depends(get_current_user)`, `try/except AppError as e: raise _app_error_to_http(e) from e`.

`location_router.py`: 동일 패턴, D-09 엔드포인트.

`world_setting_router.py`: 동일 패턴, D-10 엔드포인트. `GET /world-settings`에 `type: WorldSettingType | None = Query(None)` 파라미터 추가.
