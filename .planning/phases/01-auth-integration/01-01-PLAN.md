  - `dependencies=[Depends(require_permission("admin:users"))]`
  - query params: `page: int = Query(1, ge=1)`, `size: int = Query(20, ge=1, le=100)`
  - SQLAlchemy: `select(func.count()).select_from(User)` → total, `select(User).offset((page-1)*size).limit(size).order_by(User.created_at.desc())` → users
  - `return PaginatedUsersResponse(items=users, total=total, page=page, size=size)`

  - `status_code=status.HTTP_204_NO_CONTENT`

**0002_seed_admin_role_and_permission.py 생성** (per RESEARCH.md Pitfall 3 — DB에 admin:users permission seed 없으면 관리자도 403):

Migration 구조:
- `revision = "0002_seed_admin_role_and_permission"`
- `down_revision = "0001_initial_schema"`

`upgrade()` 함수에서 `op.execute()`로 raw SQL 사용:
1. admin role 삽입: `INSERT INTO roles (id, name, description, created_at) VALUES (gen_random_uuid(), 'admin', '관리자', now()) ON CONFLICT (name) DO NOTHING`
2. user role 삽입: `INSERT INTO roles (id, name, description, created_at) VALUES (gen_random_uuid(), 'user', '일반 사용자', now()) ON CONFLICT (name) DO NOTHING`
3. admin:users permission 삽입: `INSERT INTO permissions (id, key, description, created_at) VALUES (gen_random_uuid(), 'admin:users', '사용자 관리', now()) ON CONFLICT (key) DO NOTHING`
4. admin role과 admin:users permission 연결: `INSERT INTO role_permissions (role_id, permission_id) SELECT r.id, p.id FROM roles r, permissions p WHERE r.name = 'admin' AND p.key = 'admin:users' ON CONFLICT DO NOTHING`

`downgrade()` 함수: 역순으로 삭제 (role_permissions 먼저, 그다음 permission, role)

**주의:** 0001 migration에서 roles/permissions 테이블은 이미 생성됨. 이 migration은 데이터 seed만 담당.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Elevation of Privilege | GET/POST /admin/users/* | mitigate | `require_permission("admin:users")` 의존성으로 RBAC 체크 — DB 권한 테이블이 최종 결정권자 |
| T-01-02 | Tampering | POST /admin/users/{id}/activate | mitigate | user_id로 조회 후 존재 확인 → 없으면 404, 있으면 단일 컬럼(is_active)만 업데이트 |
| T-01-03 | Information Disclosure | GET /admin/users | accept | 이 Phase는 admin 인증 사용자만 접근 — 일반 사용자는 403. MVP 수용 |
| T-01-SC | Tampering | (해당 없음 — 신규 패키지 설치 없음) | accept | 이 Plan은 신규 패키지를 설치하지 않음 |
