### Decisions
- D-02: `tags` 컬럼 `server_default=sa.text("'[]'::jsonb")` — 문자열 단독 사용 시 PostgreSQL JSONB 타입 불일치; `sa.text()` 래핑 필수 (05-CONTEXT.md D-02)
- D-11: 단일 plan에 백엔드→SDK→프론트엔드 순서 + human action checkpoint — Task 2에서 외부 의존 발생
- D-10: SDK 재생성은 FastAPI 서버 실행 중 상태에서만 가능 — `pnpm generate:api` 전제 조건

### Surprises
- sg-next가 review → sg-learn → sg-retro로 자동 진행 — "Ready to merge? No" 결과가 워크플로우를 막지 않았다. sg-retro가 미완성 Phase에서도 실행됨.
- ruff `--fix` 가 `UP017`, `I001` 등 pre-existing 이슈도 함께 수정 — 변경 범위가 예상보다 넓어졌다.
- PLAN.md의 `read_first` 위치 정보가 Phase 4 교훈을 반영하여 라우터 3곳 누락 없이 수정됨.

### Patterns
- NovelResponse 직접 생성 패턴: 라우터 3곳 + 서비스 1곳 인라인 구성 — 새 필드 추가 시 4곳 모두 수정해야 하는 반복 패턴
- `exclude_unset=True` partial update 패턴: tags/tagline이 `None` 전송 시 기존 값 유지 — NOVEL-03 요구사항 충족

### Mistakes
- Task 3 미이행 상태에서 코드 리뷰 요청 — 백엔드만 완성된 상태에서 sg-next 호출로 review가 트리거됨. Critical 이슈 2개가 "미이행"처럼 리포트됨.
- `tagline` String(255) vs NOVEL-01 String(200) 불일치 — 계획 단계에서 감지되지 않음
