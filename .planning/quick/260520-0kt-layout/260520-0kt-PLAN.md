<interfaces>
공통 NavRail 메뉴 5개 + 설정 버튼:
- overview   → `/novels/$novelId`                  icon: BookOpen,  label: '작품 설정'
- write      → `/novels/$novelId/chapters/$chapterId/edit` (editor에서만 활성, 다른 페이지에선 단순 라벨) icon: PenLine, label: '글쓰기'
- characters → `/novels/$novelId/characters`       icon: Users,     label: '캐릭터'
- lorebook   → `/novels/$novelId/lorebook`         icon: Globe,     label: '로어북'
- storybible → (disabled, 라우팅 없음)             icon: Palette,   label: '비주얼바이블'
- 하단 고정: Settings 아이콘 (현재 inert, 동일 동작 유지)

타입:
```
export type NavId = 'overview' | 'write' | 'characters' | 'lorebook' | 'storybible';
```

TanStack Router navigate 패턴 (cast 금지):
- `navigate({ to: '/novels/$novelId', params: { novelId } })`
- `navigate({ to: '/novels/$novelId/characters', params: { novelId } })`
- `navigate({ to: '/novels/$novelId/lorebook', params: { novelId } })`
- write 항목: editor-layout에서는 disable 처리 없이 현재 chapter로 가있는 상태 유지 (다른 페이지에서는 chapterId가 없으므로 클릭 시 overview로 가거나 비활성 — **현재 동작 그대로 복제**: novel-settings/characters/lorebook 페이지에서 write를 클릭하면 어디로 가는지 원본 onClick을 그대로 옮긴다. 임의 추측 금지).

테마 토큰:
- `LIGHT_TOKENS` / `DARK_TOKENS` 객체는 기존 페이지에 있음 — 그대로 재사용해 NovelShell 루트 div의 `style`로 spread.
- NavRail은 부모의 CSS var를 읽기만 함 — 자체 토큰 spread 없음.
</interfaces>
